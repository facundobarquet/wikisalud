### Esta sección está dedicada al registro de errores, sus causas y soluciones.

Para poder optimizar la resolución de problemas es clave tener un registro de errores pasados al cual poder consultar de ser necesario.

En esta sección se deberán documentar errores conocidos, brindando la información necesaria para su entendimiento y resolución. Para lograr este cometido de manera eficiente se define el siguiente **formato**:

Cada una de las entradas en esta sección deberá tener:

* **Fecha:** fecha en la que se registra el error.
* **Autor:** miembro del equipo que registra el error, a quien se deberá remitir por más detalles.
* **Etiquetas:** palabras clave que ayudan a identificar el error rápidamente.
* **Instancia:** cuándo ocurre el error, a partir de qué acción, en qué componente.
* **Mensaje de error:** mensaje de error obtenido (de haberlo).
* **Componente que falla:** cuál o cuáles componentes causan el error.
* **Motivo del error**: razón por el cual ocurre el error.
* **Solución:** accionar necesario para resolver el error.

### Agregar o editar un registro

Para agregar un nuevo registro de error o editar uno existente se debe ingresar al perfil de administración de contenido.

* Se debe ingresar a  `url/admin`
* En el menú izquierdo ingresar a **Error Posts**.
* Si se desea crear un nuevo registro de error, seleccionar **Create New**.

![](/img/createNew.png)

* Si se desea editar un registro existente, seleccionar el menú de más opciones (tres puntos) y la opción **Edit in Admin**.

![](/img/editInAdmin.png)

* Una vez que se encuentre en el menú de edición, ingresar todos los datos según se especifica en **Formato**.

### Menú de Edición

Para escribir y dar formato al texto se cuenta con un menú de edición que permite escribir Markdown más intuitivamente. Este posee las siguientes funcionalidades:

![](/img/menuEdicion.png)(De derecha a izquierda en el menú)

* **Agregar Títulos**
* **Insertar Link**
* **Agregar Imagen** (al agregar una imagen o documento se debe cuidar que el nombre de este no contenga espacios)
* **Agregar Cita**
* **Lista**
* **Agregar Código**
* **Agregar Bloque de Código**
* **Negrita**
* **Cursiva**
* **Cambiar a edición en Markdown** (el editor simplificado puede tener errores, de persistir se recomienda editar el texto en esta opción)
